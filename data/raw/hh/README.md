# HH data layout (snapshots, deltas, state)

Этот каталог хранит артефакты ежедневного ingestion HH. Все пути ниже относительные к
`data/raw/hh/`.

## Структура

- `snapshots/` — полные снапшоты вакансий за конкретный запуск.
  - Формат имени: `snapshot_<run_id>.csv` (например `snapshot_2025-02-01T03-00-00Z.csv`).
  - Содержит все активные вакансии на момент запуска.

- `snapshots_parquet/` — Parquet-снапшоты (append-only, для аналитики).
  - Формат пути: `snapshots_parquet/date=YYYY-MM-DD/snapshot_<run_id>.parquet`.
  - Используется для долгосрочного хранения и аналитики.

- `deltas/` — дельты между текущим и предыдущим снапшотом.
  - Формат имени: `delta_<run_id>.csv`.
  - Содержит только изменения за запуск (`new`/`removed`).

- `latest.csv` — ссылка на «текущий» полный датасет.
  - Используется пайплайном как входной файл.
  - Обновляется каждым успешным запуском ingestion.

- `state.json` — метаданные последнего успешного запуска.
  - Пример полей (минимальный контракт):
    - `last_run_id` — идентификатор последнего запуска (строка).
    - `snapshot_path` — относительный путь до снапшота (строка).
    - `sha256` — контрольная сумма `latest.csv` (строка).
    - `row_count` — количество строк в `latest.csv` (int).
    - `new_count` — количество новых вакансий в дельте (int).
    - `removed_count` — количество удаленных вакансий в дельте (int).
    - `generated_at_utc` — время генерации в UTC ISO-формате (строка).
  - Допускается добавление новых полей без изменения существующих.

- `manifest.jsonl` — журнал запусков (одна строка = один run record).
  - Каждая строка — JSON-объект с метаданными run (включает поля из `state.json`
    и дополнительные служебные поля, например `duration_sec`, `status`).
  - Дополнительно сохраняется `parquet_snapshot_path` с относительным путём до Parquet.
  - Записи добавляются в конец файла; история не переписывается.

## Примечания

- Все файлы в этом каталоге, кроме `README.md`, как правило, генерируются и
  не коммитятся в репозиторий.
- Структура используется скриптом `scripts/hh_daily_refresh.py` и прод-обвязкой
  `infra/pipeline_runner.sh`.
